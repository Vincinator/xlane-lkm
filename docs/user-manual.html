User Manual¶

asgard-cli¶

Asgard CLI Entrypoint for the main click group 2 :return:

asgard-cli [OPTIONS] COMMAND [ARGS]...

eval-asgard-with-redis¶

asgard-cli eval-asgard-with-redis [OPTIONS]

Options

--input <input>¶

Path to the evaluation json file, containing messages for replication

Default

generated_data.json

--config_path <config_path>¶

Path to the configuration file. The redis leader and local IP/Port is configured in this config, as well as paths

Default

example.asgard-bench.ini

-i, --isleader¶

Set this flag if this node should be configured as leader/master node

Default

False

-r, --requests¶

If set data from the input file is read and requests are sent to the redis instance

Default

False

-p, --prepare¶

If set starts a redis environment and if isleader is NOT set, this node registers as replica at the master

Default

False

-c, --clean¶

If set cleans the dump.rdb file and kills redis-server instances on this node

Default

False

-s, --stats¶

If set reads the eval_throughput_timestamps file (path in config.ini), and prints statistics

Default

False

-a, --asgard¶

If set uses asgard for redis

Default

False

-d, --dumplogs¶

persists asgard redis logs to disk

Default

False

eval-kernel-module-no-userspace¶

asgard-cli eval-kernel-module-no-userspace [OPTIONS]

Options

--config_path <config_path>¶

Path to the configuration file. The redis leader and local IP/Port is configured in this config, as well as paths

Default

example.asgard-bench.ini

--ifindex <ifindex>¶

Uses this parameter as ifindex instead of ifindex from .ini file

Default

-1

--log_dir <log_dir>¶

Directory where the timestamp logs are downloaded to

Default

~/logs

-a, --actions <actions>¶

[Default=load,configure,clean,unload]

Comma separated list of actions to execute in specified order. The available actions are listed below:

  • start: starts the selected protocol (requires protocol flag)

  • stop: stops the selected protocol (requires protocol flag)

  • dump: dumps logs for the selected protocol (requires protocol flag)

  • configure: configures asgard and if specified also the selected protocol

  • clean: cleans the asgard instance

  • load: loads the asgard kernel module

  • unload: unloads the asgard kernel module

-p, --protocol <protocol>¶

Select the target protocol

-m, --multicast¶

configures asgard for multicast enabled features

Default

False

Instance ID of the ping pong protocol. Used for latency tests
